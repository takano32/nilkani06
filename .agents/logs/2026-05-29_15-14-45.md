## 技術メモ

### セッションファイルの形式 (`sessions/*.json`)

```json
{
  "session_start": "2026-05-29T06:14:45.000Z",
  "messages": [
    { "timestamp": "...", "channel": "C...", "role": "user",      "content": "こんにちは" },
    { "timestamp": "...", "channel": "C...", "role": "assistant", "content": "こんにちは！" }
  ]
}
```

- チャンネルごとに履歴をフィルタして OpenAI に渡す（`allMessages.filter(m => m.channel === channelId)`）
- `role` は OpenAI の `user` / `assistant` をそのまま使用

### エージェントルールの伝達構造

```
.agents/AGENTS.md   ← 実体（1箇所）
AGENTS.md           ← @.agents/AGENTS.md（Codex が読む）
CLAUDE.md           ← @.agents/AGENTS.md（Claude Code が読む）
```

### boot.sh の export について

環境変数を CLI から渡す場合（`OPENAI_API_KEY=sk-... ./boot.sh`）は export なしでも子プロセスに継承される。
スクリプト内のフォールバック値を子プロセスに渡すには `export` が必要なため追加した。

### Slack 通知 (boot.sh)

```sh
curl -s -X POST "https://slack.com/api/chat.postMessage" \
  -H "Authorization: Bearer ${SLACK_BOT_TOKEN}" \
  --data-urlencode "channel=${SLACK_NOTIFY_CHANNEL}" \
  --data-urlencode "text=${text}"
```

`--data-urlencode` による form-encoded 送信で JSON 組み立て不要。POSIX sh 互換。
